## Common Structures

- 30s: hook, one proof, one action.
- 60s: problem, process, result, lesson.
- 120s: real pain, workflow, demo beats, limitation, CTA.

## Production Package

Include:

- script
- shot list
- asset list
- caption
- thumbnail text
- publishing checklist
- missing proof list

## Reference-Driven Documentary Workflow

Use this workflow when the target is a dense explainer, documentary-style short, or business analysis video that must feel materially sourced rather than template-made.

1. Break down the reference or target style into observable rules: aspect ratio, pacing, major visual changes per minute, title rhythm, evidence shots, subtitles, sound bed, and sound-effect moments.
2. Build a visual and audio master before writing final copy: title system, subtitle rules, evidence-card pattern, data-card pattern, color treatment, transition rules, music bed, and effect timing.
3. Create an evidence table before production. Each narration beat should map to a public source, screenshot/clip candidate, on-screen text, animation note, and risk note.
4. Choose the production route honestly:
   - AE/PR master for highest polish.
   - CapCut/Jianying master for platform-speed production.
   - Remotion or HyperFrames for repeatable programmatic output.
5. Produce a draft only after the evidence table and master rules exist.
6. Run audiovisual QA before calling the draft ready for review.

## Evidence Table Format

| Beat | Voiceover | Evidence source | Visual asset | On-screen text | Motion/audio note | Risk |
|---|---|---|---|---|---|---|

Rules:

- Prefer public pages, authorized screenshots, user-owned footage, or self-created graphics.
- If evidence is missing, mark the beat as missing proof instead of inventing a card.
- Never copy full articles, paid reports, licensed music, or private platform screenshots into a public repository.

## Audiovisual QA Gate

Before delivery, check:

- Target aspect ratio and duration are correct.
- Audio and video streams are present.
- Subtitles do not cover key evidence.
- The video does not stay on plain talking-head or empty template visuals for too long.
- Each section has at least one real evidence or data visual when the script makes factual claims.
- Background music and effects support the rhythm without overpowering speech.
- A contact sheet, source table, and review notes are produced for human review.

## Voiceover-Driven Production Gate

Use this gate when a real voiceover or talking-head recording is the spine of the video. The goal is not to decorate a script with generic B-roll. The goal is to make every visual, subtitle, motion cue, music choice, and sound effect explain or emphasize the exact sentence being spoken.

Required sequence:

1. Transcribe and correct the voiceover.
2. Split the voiceover into sentence-level or semantic beats.
3. Create a director table before collecting assets.
4. Mark the job of every beat: prove, demonstrate, compare, explain, create emotion, transition, or close.
5. Build a dedicated asset package for this video. Existing assets are candidates only.
6. Create a sound table with music mood, effect type, timing, and purpose.
7. Create a rough cut to validate structure, not final polish.
8. Move to CapCut/Jianying, Premiere, After Effects, Remotion, or HyperFrames only after the beat table and asset package are ready.
9. Run final QA against the director table, not against a generic template.

Director table format:

| Beat | Voiceover | Job | Main visual | Backup visual | Motion | Audio cue | Source/risk |
|---|---|---|---|---|---|---|---|

Dedicated asset package checklist:

- Voiceover transcript and corrected subtitles.
- Source table for public pages, owned recordings, screenshots, reports, and generated assets.
- Real screen recordings or owned footage when the beat is about a real tool or process.
- Generated B-roll only where real evidence is unavailable or the beat is conceptual.
- Music plan with low-energy, emphasis, transition, and closing options.
- Sound-effect table tied to visual actions.
- Reuse justification table for any asset borrowed from a previous video.

Do not mark a version as final if the dedicated asset package is missing. Label it as a structure test, placeholder rough cut, or internal preview.

## Editing Handoff Package

When the final polish will happen in CapCut/Jianying, Premiere, or another editor, export a handoff package instead of pretending that a script-rendered preview is finished.

Include:

- Master video preview.
- Separate voice, music, and SFX stems when available.
- SRT or caption file.
- Director table.
- Source table.
- Asset manifest.
- Open issues list.
- QA notes for black frames, audio loudness, readable text, and subtitle overlap.

Known tool boundary:

- Directly generating modern CapCut/Jianying project files may be unreliable across app versions. Prefer stable media import packages unless the current editor format has been verified.
- Programmatic renderers are useful for structure, subtitles, overlays, and repeatable packaging, but the final polish gate should still check pacing, readability, source fit, music, and human review notes.

## Preview-First Production States

Use this when the video is iterated through browser preview, rendered MP4, and optional editor polish.

State labels:

1. `script-ready`: topic, claim boundary, timeline script, and source table exist.
2. `preview-review`: a browser or low-cost preview can be reviewed by timecode.
3. `mp4-rendered`: an MP4 exists, but it has not passed final QA.
4. `qa-passed`: duration, audio, subtitles, visual proof, and text readability have been checked.
5. `editor-polish-ready`: stable media, captions, source table, and open issues are packaged for CapCut/Jianying, Premiere, or another editor.
6. `publish-ready`: final human review is complete and platform-specific captions, cover, and export settings are confirmed.
7. `published`: platform evidence exists.

Rules:

- Do not skip directly from preview to publish-ready.
- Review previews by timecode, not by vague comments.
- Keep engineering preview, rendered MP4, QA-passed file, editor handoff, and platform publication as separate deliverables.
- If a user voiceover is missing, label the project as preproduction or structure preview.
- If customer, account, or private footage appears in a draft, keep it private and record the redaction requirement before public packaging.
